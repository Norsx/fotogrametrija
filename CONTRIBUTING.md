# Contributing

## Branch policy

- Use feature branches (`task/*`, `fix/*`, `docs/*`).
- Never push directly to `main`.

## Pull request expectations

- Scope PR to one task.
- Explain what changed, why, and key risks.
- Link relevant review output from `reviews/`.

## AI-assisted changes

- AI-generated changes are allowed but must be reviewed.
- Keep generated output reproducible and traceable.
- Do not commit secrets, credentials, or private datasets.
